## 8. Known bugs / debt

### High priority (blocks demo polish)

1. **Only 33/~3000 task_assignees imported.** Sync log shows hundreds of `duplicate key value violates unique constraint "uq_task_assignees_task_role"` errors. Cause: many Odoo tasks have multiple `user_ids`, importer inserts each as `role_type='agent'`, unique `(task_id, role_type)` rejects 2nd+ insert. **Fix**: dedupe `user_ids` per task in `importTasks` before insert, OR distribute the duplicates across role slots heuristically. File: `src/lib/odoo/importer.ts` around the `task_assignees` insert block.

2. **Old seeded departments overlap with new sl-* tree** in `/organization/chart`. Cleanup migration: `delete from departments where organization_id = (...) and slug not like 'sl-%' and external_source is null;` — **but verify no FKs from elsewhere first.**

### Medium

3. `/projects` still reads from live Odoo via `listLiveProjectsPaged`. Needs the same Supabase-mirror refactor that `/tasks` got. File: `src/app/(dashboard)/projects/page.tsx`.

4. Several pre-existing TypeScript / lint errors throughout the repo (e.g. `src/app/(dashboard)/escalations/_actions.ts:34` zod import, `src/app/(dashboard)/projects/[id]/page.tsx` implicit `any`). These were NOT introduced this session — verify before "fixing."

5. **Imported clients = 4** — most Odoo projects have no `partner_id`, all funnel to the "عميل غير محدد" placeholder. Real fix: source client name from project name when no partner is set, or pull from a different Odoo association.

### Low

6. Renewal categories ("🔵Renewal Social Media", "🟠Renewal SEO", etc.) imported as separate `services` rows. Per owner directive, renewals are new projects — but the duplicated services may confuse the UI. Consider an `is_renewal` boolean instead of duplicate categories.

7. `project.category` in Odoo = `services` in our schema, but the existing seeded services (`social-media-management`, `seo`, `media-buying`) sit alongside the imported ones with different slugs. Pick one source of truth.
